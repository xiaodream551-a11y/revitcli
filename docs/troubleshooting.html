Troubleshooting

Start with revitcli doctor — it checks configuration, server connection, and profile detection.

"Revit is not running or plugin is not loaded"

This means the CLI cannot connect to the Revit add-in HTTP server.

Checklist

Check Fix
Is Revit open? Start Revit and open a project.
Is the add-in installed? Check %APPDATA%\Autodesk\Revit\Addins\2026\RevitCli.addin exists.
Is the DLL path correct? Open the .addin file and verify the <Assembly> path points to the actual DLL.
Was Revit restarted after install? Revit loads add-ins at startup. Close and reopen Revit.
Is the port blocked? Check firewall. The add-in uses port 17839 by default (tries 10 ports).
Is server.json stale? Delete ~/.revitcli/server.json and restart Revit.

"No .revitcli.yml found"

The CLI looks for .revitcli.yml starting from your current directory, walking up to the filesystem root.

Fix

# Copy a starter profile to your project root
cp profiles/general-publish.yml .revitcli.yml

# Or specify explicitly
revitcli check --profile /path/to/.revitcli.yml

"Check set 'xxx' not found in profile"

Your profile doesn't have a check set with that name. Run revitcli doctor to see available check sets. 

# List what's in your profile
revitcli doctor

# Run a specific check set
revitcli check default
revitcli check pre-issue

"Unknown audit rule(s): xxx"

You referenced a rule that doesn't exist. Available rules:

revitcli audit --list

Current rules: naming, room-bounds, level-consistency, unplaced-rooms, views-not-on-sheets, imported-dwg, in-place-families, duplicate-room-numbers, room-metadata, sheets-missing-info.

"Parameter 'xxx' has N definitions on this element"

The element has multiple parameters with the same name (e.g. instance vs type parameter). Disambiguate with the [N] suffix: 

# Query to see which is which
revitcli query --id 12345 --output json

# Set the second one
revitcli set --id 12345 --param "Comments [2]" --value "Updated"

"Parameter 'xxx' is read-only"

This parameter cannot be modified via the API. Common read-only parameters include system-calculated values like Area, Volume, and Perimeter.

"Cannot convert 'abc' to integer/number"

The value you're trying to set doesn't match the parameter's data type.

Storage type Expected value Example
String Any text --value "Hello"
Integer Whole number --value "42"
Double Number (in display units) --value "3000" for mm
ElementId Element ID number --value "337596"
Use --dry-run to test value compatibility before applying changes.

Export produces duplicate pages

When exporting PDF without specifying --sheets, all printable views are included. For simple projects this may produce duplicate-looking pages (e.g. multiple floor plans of the same level).

Fix

# Export specific sheets only
revitcli export --format pdf --sheets "A101" --output-dir ./exports

# Or use a profile with explicit sheet selection
# In .revitcli.yml:
exports:
  my-pdf:
    format: pdf
    sheets: ["A101", "A102"]
    outputDir: ./deliverables/pdf

"--stdin: no input received"

PowerShell pipe + dotnet run doesn't reliably pass stdin. Use --ids-from instead: 

# Save query results to file first
revitcli query doors --output json > doors.json

# Then use the file
revitcli set --ids-from doors.json --param "Mark" --value "OK" --dry-run

Profile parse error

Your .revitcli.yml has a syntax error. Common issues:

Error Cause
failOn must be 'error' or 'warning' Typo in failOn value (e.g. errors)
severity must be error/warning/info Typo in severity (e.g. warnng)
Circular profile inheritance Two profiles extend each other
YAML syntax error Indentation or formatting issue

Add-in version mismatch

If revitcli status shows an old add-in version, the deployed DLL is outdated.

Fix

# 1. Close Revit completely (check Task Manager)
# 2. Clean old files
Remove-Item "$env:APPDATA\Autodesk\Revit\Addins\2026\RevitCli*" -Force
# 3. Rebuild and redeploy
dotnet publish src/RevitCli.Addin -c Release -f net8.0-windows -p:RevitYear=2026
Copy-Item src\RevitCli.Addin\bin\Release\2026\publish\* "$env:APPDATA\Autodesk\Revit\Addins\2026\" -Force
# 4. Start Revit and verify
revitcli status
