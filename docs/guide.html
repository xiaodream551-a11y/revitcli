Core Workflows

RevitCli has three core workflows. Each one replaces a manual process that BIM teams do every week.

Workflow 1: Model Quality Check

Run standardized checks against your Revit model and generate reports.

Basic check

revitcli check

Runs the default check set from your .revitcli.yml profile. Exits with code 1 if any issues exceed the failOn threshold.

Named check set

revitcli check pre-issue

Runs a specific check set. Use stricter sets for milestones, relaxed sets for daily work.

Generate reports

# HTML report (send to BIM manager)
revitcli check --report check-report.html

# JSON report (feed to CI/scripts)
revitcli check --output json

# Both: save HTML file + print table to console
revitcli check --report report.html --output table

Track progress over time

RevitCli automatically saves check results. Run it again to see what changed: 

revitcli check
# ... fix some issues ...
revitcli check
# Output:
# Check 'default': 8 passed, 1 failed
# vs previous: 0 new, 3 resolved, 5 unchanged
#   [RESOLVED] room-bounds: Room 'Kitchen' has zero area
#   [RESOLVED] naming: View 'Section 1' appears to use a default name

Suppress known issues

Real projects have legacy issues you can't fix right now. Add suppressions to your profile: 

# In .revitcli.yml
checks:
  default:
    failOn: error
    auditRules:
      - rule: naming
    suppressions:
      - rule: naming
        reason: "Legacy views from template, will fix in Phase 2"
        expires: "2026-06-01"
      - rule: required-parameter
        category: doors
        parameter: Fire Rating
        elementIds: [337596, 337601]
        reason: "Placeholder doors"

Suppressed issues don't count toward pass/fail and show a separate count.

Workflow 2: Deliverable Publishing

Export consistent deliverable packages with pre-checks.

Preview first

revitcli publish --dry-run

Shows what would be exported without actually exporting.

Publish with quality gate

revitcli publish

If a precheck is defined in the pipeline, it runs first. Publishing is aborted if the precheck fails.

Named pipelines

# Default pipeline (e.g. DWG + PDF)
revitcli publish

# Full pipeline (e.g. DWG + PDF + IFC)
revitcli publish full

Direct export (without profile)

# Export DWG
revitcli export --format dwg --sheets all --output-dir ./exports

# Export PDF for specific sheets
revitcli export --format pdf --sheets "A1*" --output-dir ./exports

# Export IFC (whole model)
revitcli export --format ifc --output-dir ./exports

# Export specific views
revitcli export --format dwg --views "Level 1" --output-dir ./exports

Workflow 3: Parameter Query and Modification

Find elements and batch-edit parameters.

Query elements

# All walls
revitcli query walls

# With filter
revitcli query doors --filter "Fire Rating="

# Specific element
revitcli query --id 337596

# Output formats
revitcli query rooms --output json
revitcli query rooms --output csv > rooms.csv

Modify parameters

# Preview changes first (always!)
revitcli set walls --param "Mark" --value "W-01" --dry-run

# Apply changes
revitcli set walls --param "Mark" --value "W-01"

# With filter
revitcli set doors --filter "Fire Rating=" --param "Fire Rating" --value "60min"

# Single element
revitcli set --id 337596 --param "Comments" --value "Reviewed"
Always use --dry-run first. The set command modifies your Revit model. Changes are wrapped in a single transaction (all-or-nothing), but there is no undo from the CLI.

Batch from file

# Export element IDs to file
revitcli query doors --filter "Fire Rating=" --output json > doors.json

# Apply changes from file
revitcli set --ids-from doors.json --param "Fire Rating" --value "60min" --dry-run

Other Commands

Command Description
revitcli status Show Revit version, add-in version, document, capabilities
revitcli doctor Diagnose setup: config, server, connection, profile detection
revitcli audit --list List all available audit rules
revitcli audit --rules naming,room-bounds Run specific audit rules directly (without profile)
revitcli config show Show CLI configuration
revitcli config set defaultOutput json Change CLI defaults
revitcli batch commands.json Execute commands from a JSON file
revitcli completions bash Generate shell completions
revitcli -i Interactive REPL mode
