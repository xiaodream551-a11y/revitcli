What is RevitCli?

RevitCli is a command-line tool that runs model quality checks and publishes deliverables for Autodesk Revit projects. It turns your BIM standards into executable, repeatable, version-controlled workflows.

revitcli check

Run audit rules, required parameter checks, and naming patterns from a shared profile. Generate HTML/JSON reports. Diff against previous runs.

revitcli publish

Pre-check gate + batch export pipeline. DWG, PDF, IFC with consistent settings every time. One command, no guesswork.

revitcli query + set

Query elements by category/filter, output as JSON/CSV. Batch-modify parameters with dry-run preview and transaction safety.

Quick Start (5 minutes)

Install the CLI

dotnet tool install --global RevitCli

Or build from source: dotnet publish src/RevitCli -c Release -o ./publish

Deploy the Revit Add-in

dotnet publish src/RevitCli.Addin -c Release
# Close Revit, then copy files to:
# %APPDATA%\Autodesk\Revit\Addins\2026\

Or use the installer: powershell -ExecutionPolicy Bypass -File install.ps1

Verify connection

revitcli doctor

Should show green checks for Revit version and document.

Copy a starter profile

# Choose one:
cp profiles/architectural-issue.yml .revitcli.yml
cp profiles/interior-room-data.yml .revitcli.yml
cp profiles/general-publish.yml .revitcli.yml

Run your first check

revitcli check
revitcli check --report report.html   # open in browser

Why RevitCli?

Without RevitCli With RevitCli
Manually browse schedules, views, sheets before issue revitcli check pre-issue in 10 seconds
Export DWG/PDF one by one with different settings revitcli publish with consistent presets
Standards are in a Word doc nobody reads Standards are in .revitcli.yml, enforced automatically
"Did we check the model?" — "I think so" HTML report + journal log + webhook notification
Fix 200 door fire ratings one by one revitcli set doors --filter "Fire Rating=" --param "Fire Rating" --value "60min"

Audit Rules

Rule Category What it checks
room-bounds Rooms Rooms with zero area (not enclosed)
unplaced-rooms Rooms Rooms not placed in model
duplicate-room-numbers Rooms Multiple rooms sharing same number
room-metadata Rooms Missing room number or default name
level-consistency Levels Duplicate elevations
naming Naming Default view names (Section 1, 3D View 2)
views-not-on-sheets Delivery Printable views/schedules not on any sheet
sheets-missing-info Delivery Sheets with no number or empty content
imported-dwg Health Imported (not linked) CAD files
in-place-families Health In-place families that should be loadable

Plus profile-driven: required-parameter and naming-pattern (custom regex).
