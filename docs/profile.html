Profile Reference

The .revitcli.yml file defines your project's BIM standards as executable configuration. Place it in your project root and commit to version control.

Full Schema

version: 1

# Inherit from a parent profile (optional)
# Child replaces entire named entries; defaults merge field-by-field.
extends: ../company-standards/base.revitcli.yml

defaults:
  outputDir: ./deliverables          # default export directory
  notify: https://hooks.slack.com/...  # webhook URL (HTTPS only)

checks:
  <name>:
    failOn: error | warning          # exit code threshold
    auditRules:
      - rule: <rule-name>
    requiredParameters:
      - category: <category>
        parameter: <param-name>
        requireNonEmpty: true | false
        severity: error | warning | info
    naming:
      - target: views | sheets | <category>
        pattern: <regex>
        severity: error | warning | info
    suppressions:
      - rule: <rule-name>
        category: <optional>         # narrow by category
        parameter: <optional>        # narrow by parameter
        elementIds: [id1, id2]        # narrow by element (optional)
        reason: "explanation"          # why this is suppressed
        expires: "2026-12-31"          # auto-expire date (optional)

exports:
  <name>:
    format: dwg | pdf | ifc
    sheets: [pattern1, pattern2]     # sheet name/number patterns
    views: [pattern1, pattern2]      # view name patterns
    outputDir: ./path                # overrides defaults.outputDir

publish:
  <name>:
    precheck: <check-name>           # run this check set first
    presets: [export1, export2]      # export presets to run

Section Details

version

Always 1. Reserved for future schema changes.

extends

Path to a parent profile (relative to this file). Inheritance rules:

Section Merge behavior
defaults Field-level merge (child overrides individual fields, inherits the rest)
checks Merge by name. Child replaces the entire named check definition.
exports Merge by name. Child replaces the entire named export preset.
publish Merge by name. Child replaces the entire named pipeline.
Named entries are full-object replacement, not deep merge. If a child overrides checks.default, it must include all rules — the parent's rules are not inherited into that check set.

defaults

Field Type Description
outputDir string Default export directory. Paths are relative to the profile file.
notify string Webhook URL. Must be HTTPS. Receives JSON POST after check/publish.

checks.<name>

failOn

Value Behavior
error Exit 1 only if errors found. Warnings pass.
warning Exit 1 if errors or warnings found. Stricter.

auditRules

List of built-in rules to run. See Audit Rules for the full list.

requiredParameters

Field Type Description
category string Element category (e.g. doors, walls, rooms). English or Chinese.
parameter string Parameter name to check.
requireNonEmpty bool If true, parameter must exist AND have a non-empty value.
severity string error, warning, or info.

naming

Field Type Description
target string views, sheets, or a category name.
pattern string Regex pattern. Elements whose names do NOT match are flagged.
severity string error, warning, or info.

suppressions

Suppress known issues so they don't count toward pass/fail.

Field Required Description
rule Yes Rule name to suppress (e.g. naming, required-parameter).
category No Only suppress issues mentioning this category.
parameter No Only suppress issues mentioning this parameter.
elementIds No Only suppress these specific element IDs.
reason No Human-readable explanation (shows in reports).
expires No Date (YYYY-MM-DD) after which suppression is ignored.

exports.<name>

Field Type Description
format string dwg, pdf, or ifc.
sheets string[] Sheet patterns. Use all for everything, or wildcards like A1*.
views string[] View name patterns. Not used for IFC (whole-model export).
outputDir string Export directory. Overrides defaults.outputDir.

publish.<name>

Field Type Description
precheck string Name of a check set to run before exporting. Aborts on failure.
presets string[] List of export preset names to run sequentially.

Webhook Payload

check

{
  "type": "check",
  "check": "default",
  "passed": 8,
  "failed": 1,
  "suppressed": 2,
  "issueCount": 3,
  "status": "failed",
  "timestamp": "2026-04-04T12:00:00Z"
}

publish

{
  "type": "publish",
  "pipeline": "default",
  "succeeded": 2,
  "failed": 0,
  "presets": ["dwg", "pdf"],
  "status": "passed",
  "timestamp": "2026-04-04T12:00:00Z"
}

Exit Codes

Code Meaning
0 All checks passed (or below failOn threshold)
1 Checks failed, or command error
