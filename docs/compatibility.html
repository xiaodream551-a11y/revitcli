Output Contracts

The following output formats are locked and will not change without a major version bump:

  • query --output json — element array with id (long), name, category, typeName, parameters
  • check --output json — object with check, passed, failed, suppressed, timestamp, issues[]
  • check HTML report — dark theme, summary cards, issue table
  • Exit codes — 0 = pass, 1 = fail/error
  • Webhook payload — type, status, timestamp always present
  • Journal format — JSONL, one object per line

Profile Schema

Profile schema is at version: 1. Future schema changes will increment this number. RevitCli will reject profiles with unknown versions and provide migration guidance.

Breaking Changes Policy

  • Major version (1.x → 2.x): may change output contracts, profile schema, command behavior
  • Minor version (0.x → 0.y): additive changes only, no breaks to stable commands
  • Patch version (0.x.y): bug fixes only

Known Limitations

  • --stdin pipe does not work reliably with PowerShell + dotnet run. Use --ids-from.
  • --ids-from has a serialization issue with some add-in deployments. Verify with revitcli status that add-in version matches CLI version.
  • Export is synchronous — no real-time progress tracking.
  • PDF export defaults to all printable views (may produce duplicate pages on simple projects).
  • Profile inheritance is full-object replacement for named entries (not deep merge within check/export/publish definitions).
